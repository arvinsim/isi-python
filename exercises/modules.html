RO13

ROT13 ("rotate by 13 places", sometimes hyphenated ROT-13) is a simple letter substitution cipher that replaces a letter with the letter 13 letters after it in the alphabet. ROT13 is a special case of the Caesar cipher, developed in ancient Rome.

In the basic Latin alphabet, ROT13 is its own inverse; that is, to undo ROT13, the same algorithm is applied, so the same action can be used for encoding and decoding. The algorithm provides virtually no cryptographic security, and is often cited as a canonical example of weak encryption.

An example

What to do

Create a package named "lib". In the package, create a module named "functions". The module should contain the class named "ROT13". The class should take a string parameter, which is required and will be reversed as you assign it. Define a method called "render", this will then display the rotated string output. You should be able to create an algorithm to satisfy the ROT13 substitution cipher.

What to do

Name your main module as "rot13.py".

Sample Code


    r = ROT13('ehugen lnqugevo lccnu')
    r.render() # deciphered output

Team Pythonistas

In basketball, a player’s statistics like points, rebounds, and assists are recorded for analysis. Since the typical basketball game is fast paced, they normally record what a player does (e.g. rebound, make an assist, miss a free throw) by simply listing each action in a single statline per player. At the end of the game, each player’s performance is recorded by processing the player’s stats.

For example, a player’s statline may look like this: 2XYRTXRA3YX012ARY

The player’s statistics can then be determined using the following legend:

Stat Meaning
1 made a 1 point shot or free throw
0 missed a 1 point shot or missed a free throw
2 made a 2 point shot
3 made a 3 point shot
Stat Meaning
X missed a 2 point shot
Y missed a 3 point shot
A made an assist
R made a rebound
T committed a turnover

In the above example (2XYRTXRA3YX012ARY), based on the player’s statline, the player

  • made one freethrow and missed one
  • made two and missed three 2-point shots
  • made one and missed three 3-pt shots
  • made two assists
  • caught three rebounds
  • committed one turnover
Points 8
Freethrows 50% (1/2)
2-point shots 40% (2/5)
3-point shots 50% (1/4)
Assists 2
Rebounds 3
Turnovers 1

You must create a program that will read the players’ statline from a text file and display out the statistics.

Input specifications

A simple text file shall be source of all input data, with the filename basketball.txt

Input specifications

The input file contains several lines. Each line contains the player’s jersey number followed by a single space and then the player’s statline, as described in above stats example (see 2XYRTXRA3YX012ARY).

Output Specifications

The output should display the player's statline described above. The first line of each player’s statistics displays the jersey number of the player, preceded by the word "PLAYER". The second line shows the number of points preceded by the word "Points".

Output Specifications

The next three lines will display the shooting percentages (preceded by "Freethrows", "“2-point shots", and "“3-point shots"), indented by a tab or four (4) spaces (whichever you prefer). The last three lines indicate the rest ("Assists", "Rebounds", "Turnovers").

What to do

Create two classes.

What to do

The Shot class will contain instance variables namely, "made" and "missed". This class will also need to make percentage calculations and will sum up the total shot attempts (made and missed). The class should make use of the object representation magic method (__repr__) to display the object output. You will also need the math function "ceil" specifically to compute the shot point percentage (use the "from X import Y" variation).

What to do

The Basketball class is where the players statlines will be parsed based on the text file input provided. You will only need to provide the "display_stats" method to display the player statline. It would be up to your discretion to create the necessary methods to achieve the implementation. Again, the Basketball class could contain several methods as needed.

What to do

(Optional) To normalize the program, you can create several classes (ie, Shot, Stats, Player, Basketball) where the Shot and Basketball classes are required.

This approach is recommended since it makes the logic easier to reason out.

Required lines of code to run the program


    b = Basketball('data/basketball.txt')
    b.display_stats()
