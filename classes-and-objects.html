Object Oriented Programming

What is Object Oriented Programming(OOP)?

A programming paradigm based on the concept of "objects", which are data structures that contain:

  • data in the form of fields, often known as attributes or attributes
  • code in the form of procedures, often known as methods or class functions

Objects can sometimes correspond to things found in the world.

  • A graphics program may have objects such as "circle," "square," "menu".
  • An online shopping system might have objects such as "shopping cart," "customer," and "product."

Sometimes objects represent more abstract entities

  • an object that represents an open file
  • an object which provides the service of translating measurements from U.S. customary to metric.

Classes

Defining a Class

Classes are defined by using the class keyword


    class OldStyleClass():
        ...

    class NewStyleClass(object):
        ...
                

    class OldStyleClass():
        pass

    class NewStyleClass(object):
        pass
                

    class OldStyleClass():
        osp = "an attribute of a old style class"

    class NewStyleClass(object):
        nsp = "an attribute of a new style class"

Just like functions, class definitions do nothing by themselves and have their own scope.

Attributes

In the previous example, there was a variable named nsp


    class NewStyleClass(object):
        nsp = "an attribute of a new style class"

This is what you call an attribute

  • Attributes belong to the scope of the class
  • The value in the declaration is their default value
  • Just like variables, they can be of any data type

Think of attributes as variables that describe an object.


    class Person(object):
        """A Person. Persons have these attributes

            age: a number representing the age of person
            sex: a string representing the sex of the person
            location: a dictionary representing a set of data related to
                    the Person's location
        """
        age = 18
        sex = "female"
        location = {
            'city': 'Cagayan de Oro',
            'street': 'Claro M. Recto Avenue',
            'country': 'Philippines'
        }

There are two types of attributes:

  • class
  • instance

What is a class?

A user-defined prototype for an object that defines a set of attributes that characterize any object of the class.

Put simply, classes are how you implement and apply the concept of objects.

Think of classes as "blueprints" for your objects

In our last example, the class Person is a blueprint for making Person objects


    class Person(object):
        age = 18
        sex = "female"
        location = {
            'city': 'Cagayan de Oro',
            'street': 'Claro M. Recto Avenue',
            'country': 'Philippines'
        }

Objects

Creating Objects

Now that we know how to define our classes or create our "blueprints", we can now move on to creating actual objects

The term for creating an object is instantiation.

Creating an object == Instantiating an object

When we instantiate, we create a new object. That new object is an instance of the class you created it from.


    class Person(object):
        pass

    john = Person()

The syntax is


    Person() #<name_of_class>()

Warning: Instantiating an object looks exactly like a function call.

Python Programmers use the convention below to differentiate between the two:

  • Classes start with an Uppercase() letter
  • Functions start with a lowercase() letter

    def doppelganger():
        pass

    class Doppelganger(object):
        pass

    print doppelganger()
    print Doppelganger()

Each instance is unique


    jane = Person()
    print john == jane # False

The type of an instance is that of its class. Basically, a class is its own datatype.


    print type(john) # <class '__main__.Person'>

More on attributes

Remember this code?


    class Person(object):
        age = 18
        sex = "female"
        location = {
            'city': 'Cagayan de Oro',
            'street': 'Claro M. Recto Avenue',
            'country': 'Philippines'

Let's instantiate a new person


    jane = Person()

Getting attributes

Suppose we want to get information about that person. In this case, we want to get the attributes. We do it like this:


    jane.age                    # don't do anything with the property
    print jane.age              # print the property
    a_persons_age = jane.age    # assign the property

Setting attributes

Let's try instantiating another person


    john = Person()

Assuming John is a middle aged male, the attributes doesn't seem to describe him correctly


    print john.age # 18
    print john.sex # female

We can change the attributes value by setting them. The syntax is shown below:


    john.age = 35
    john.sex = 'male'

If we print the attributes, we see that they have changed


    print john.age # 35
    print john.sex # 'male'

Note: You can add new attributes to an object after instantiation.


    john.first_name = "John"
    john.last_name = "Doe"
    print john.first_name
    print john.last_name

However, that is not considered a good habit. Always declare your attributes in the class definition if possible


    class Person(object):
        first_name = ""
        last_name = ""
        age = 18
        sex = "female"
        location = {
            'city': 'Cagayan de Oro',
            'street': 'Claro M. Recto Avenue',
            'country': 'Philippines'
        }

Methods

If attributes defines what objects are, methods defines what objects do

Methods are just functions for a class. As such, they are just like functions except for some differences

1.) They have a mandatory first parameter called self


    class Vehicle(object):
        def move_forward(self):
            pass

2.) They are called like attributes


    class Vehicle(object):
        def move_forward(self):
            pass

    car = Vehicle()
    car.move_forward()

Think of it this way:

  • attributes -> object variables
  • methods -> object functions

Self

Here are some important things to remember about the self keyword

1.) Even though self is the first parameter, arguments passed to the method correspond only to the second parameter and up


    class Vehicle(object):
        def turn_on(self):
            pass

    car = Vehicle()
    car.turn_on()
                

    class Vehicle(object):
        def move(self, direction="left", distance=0):
            pass

    car = Vehicle()
    car.move(direction="right", distance=100)

2.) self refers to the current instance of the object. You can access the current instance's attributes via self


    class Vehicle(object):
        def get_full_name(self):
            return "{} {}".format(self.manufacturer, self.name)

    skyline = Vehicle()
    skyline.manufacturer = "Nissan"
    skyline.name = "Skyline GT-R"
    print skyline.get_full_name() # Nissan Skyline GT-R

    impreza = Vehicle()
    impreza.manufacturer = "Subaru"
    impreza.name = "Impreza WRX"
    print impreza.get_full_name() # Subaru Impreza WRX

3.)You can access other methods via self too.


    class Vehicle(object):
        def get_full_name(self):
            return "{} {} {}".format(
                self.manufacturer,
                self.name,
                self.get_body_type())

        def get_body_type(self):
            return "grand tourer"

    skyline = Vehicle()
    skyline.manufacturer = "Nissan"
    skyline.name = "Skyline GT-R"
    print skyline.get_full_name() # Nissan Skyline GT-R ground tourer

Constructors

There is a special method for classes that you can implement that will automatically execute once you instantiate an object

This method is called a constructor

The syntax for implementing it is shown below


    class Vehicle(object):
        def __init__(self, ...): # This is the signature of a constructor
            pass

If we try to instantiate an object, the constructor will automatically execute


    class Vehicle(object):
        def __init__(self): # This is the signature of a constructor
            print "Creating a Vehicle object"

    car = Vehicle() # "Creating a Vehicle object" will be printed

Just like functions, constructors can be set with whatever parameters you want.


    class Vehicle(object):
        def __init__(self, name="", manufacturer="", body_style=""):
            print """
            Creating a {} {} with a {} body style
            """.format(manufacturer, name, body_style)

To pass arguments to it, pass it in the instantiation call


    class Vehicle(object):
        def __init__(self, name="", manufacturer="", body_style=""):
            print """
            Creating a {} {} with a {} body style
            """.format(manufacturer, name, body_style)

    # Prints "Creating a Toyota Sprinter Trueno with
    # a coupe body style
    car = Vehicle(
        manufacturer="Toyota",
        body_style="coupe",
        name="Sprinter Trueno"
    )

Class and Instance Attributes

Class attributes are defined below the class declaration


    class Vehicle(object):
        color = "red"

Class attributes are called by using the class name instead of the object instance


    car = Vehicle()
    print Vehicle.color

Instance attributes are defined in the constructor using self


    class Vehicle(object):
        def __init__(self):
            self.manufacturer = "Ferrari"

Instance attributes are called using the instance name


    car = Vehicle()
    print car.manufacturer

Class attributes are shared between all instances of that class while instance attributes are not.


    car1 = Vehicle()
    car2 = Vehicle()

    Vehicle.color = "blue"
    car1.manufacturer = "Maserati"
    car2.manufacturer = "Lamborghini"

    print car1.color
    print car1.manufacturer
    print car2.color
    print car2.manufacturer

The order of precedence when variables are accessed.

  1. Instance Attributes of that attribute name
  2. Class Attributes of that attribute name

    class Vehicle():
        color = "red"

    car1 = Vehicle()
    car2 = Vehicle()

    print car1.color
    print car2.color
