Definition

What is Inheritance?

Inheritance is when a class is based on another class, using the same implementation (inheriting from an object or class) specifying implementation to maintain the same behavior.
Inheritance is used to indicate that one class will get most or all its features from another class

Composition vs Inheritance

Inheritance

Basics

Ever wonder what that "object" argument in a class declaration means? 

    class AClass(object):
        pass
The object parameter that you pass to a Python class is actually a built-in class in Python.

    # "object" is actually a built-in class!
    class AClass(object):
        pass
Passing the "object" class as an argument to a class means that the class is inheriting from the "object" class

    # This inherits the built-in 'object' class
    class ChildOfObjectClass(object):
        pass
How do we make a class inherit another class?
To make an object inherit another class, simply replace the "object" parameter with the name of the class you want to inherit from

    class Parent(object):
        pass

    # We are declaring that Child inherits from Parent
    class Child(Parent):
        pass

Parent and Child relationship
Parent classes(or Superclasses) are the classes we inherit from

    # Parent class
    class Vehicle(object):
        pass

    class Car(Vehicle):
        pass
Child classes(or Subclasses) are the classes that inherit from one or more Parent class

    class Vehicle(object):
        pass

    # Child class
    class Car(Vehicle):
        pass

Inherited Attributes and Methods

A child class inherits all the attributes and methods of the parent class 

    class Parent(object):
        an_attribute = "parent attribute"

        def a_method(self):
            print "parent method"

    class Child(Parent):
        pass

    c = Child()
    print c.an_attribute # "parent attribute"
    c.a_method() # "parent method"

Exceptions to this rule are methods and attributes that start with a double underscore(__) don't get inherited


    class Parent(object):
        __an_attribute = "parent attribute"

        def __a_method(self):
            print "parent method"

    class Child(Parent):
        pass

    c = Child()
    print c.__an_attribute # Error!
    c.__a_method # Error!

Overriding

What happens when an attribute or a method in a child class has the same name as the parent class?

The answer is that the child class will "override" the attribute or method of the parent class

Overriding means that the child's implementation of the attribute or method will replace the implementation of the parent for all instances of that child class

Here is an example of the child overriding some of the parent's attributes


    class Parent(object):
        message = "Hello There!"
        age = 40

    class Child(Parent):
        age = 8
        gender = "male"

    c = Child()
    print c.message
    print c.age # Overriden!
    print c.gender

Here is an example of the child overriding a parent's methods


    class Parent(object):
        def get_hobbies(self):
            return ['yoga', 'cooking', 'photography']

    class Child(Parent):
        def get_hobbies(self):
            return ['eating', 'playing', 'pooping']

    c = Child()
    print c.get_hobbies()
It doesn't matter if the parameters are different. If a child attribute or method has the same name as the parent, it will replaced that attribute or method.

    class Parent(object):
        def receives(self):
            print "I am too old for this $hIt! - Grumpy Parent"

    class Child(Parent):
        def receives(self, item=None):
            if item == "blanket":
                print "ba ba, blankie"
            elif item == "pacifier":
                print "binkie wuv"
            else:
                print "muh muh boo boo"

    c = Child()
    # .receives() is overriden to an implementation that
    # accepts 1 argument
    c.receives("pacifier")

Extending

We already have discussed how to override attributes and methods.
However, sometimes we don't want the child class to override the some of the parent's methods.
We might want to reuse those methods because they can still help.
In that case, we want to "extend" the class
To extend a class, we need to use the super() built-in function
The super() built-in function allows us to call attributes and methods from the parent in the child class
Here is the typical way the super() built-in function is used.

    class Parent(object):
        def method(self, arg):
            print "This is the arg: {}".format(arg)

    class Child(Parent):
        def method(self, arg):
            arg = "{}(called by Child)".format(arg)
            super(Child, self).method(arg)
Let's breakdown the relevant code

    # super(Child, self) evaluates to the parent class
    # (which is named Parent)

    # super(Child, self).method(arg) calls the method named
    # "method" in the Parent Class
                

    c = Child()

    # This prints out "This is the arg: foobar(called by Child)"
    c.method("foobar")
Here is an example of a child class calling a parent attribute

    class Parent(object):
        message = "Don't worry child, Mommy is here"

    class Child(Parent):
        def consult_parent(self):
            return super(Child, self).message

    c = Child()
    print c.consult_parent()
