Introduction

Definition

What is Inheritance?

Inheritance is the practice of passing on property, titles, debts, rights and obligations upon the death of an individual.
Not really the best definition to use
Let's try again, this time in biology using the definition of heredity
Heredity is the passing of traits from parents to their offspring. You can say that the offspring inherits the traits from the parents.
Much better and closer to the concept that we want
Finally, the definition of inheritance in programming, paraphrased for simplicity
Inheritance is when a class will get or inherit some of its implementation from other class or classes. This includes attributes and methods.

Basics

Ever wonder what that "object" argument in a class declaration means? 

    class AClass(object):
        pass
The object parameter that you pass to a Python class is actually a built-in class in Python.

    # "object" is actually a built-in class!
    class AClass(object):
        pass
Passing the "object" class as an argument to a class means that the class is inheriting from the "object" class

    # This inherits the built-in 'object' class
    class ChildOfObjectClass(object):
        pass
How do we make a class inherit another class?
To make an object inherit another class, simply replace the "object" parameter with the name of the class you want to inherit from

    class Parent(object):
        pass

    # We are declaring that Child inherits from Parent
    class Child(Parent):
        pass

Parent and Child relationship
Parent classes(or Superclasses) are the classes we inherit from

    # Parent class
    class Vehicle(object):
        pass

    class Car(Vehicle):
        pass
Child classes(or Subclasses) are the classes that inherit from one or more Parent class

    class Vehicle(object):
        pass

    # Child class
    class Car(Vehicle):
        pass

Inherited Attributes and Methods

A child class inherits all the attributes and methods of the parent class 

    class Parent(object):
        an_attribute = "parent attribute"

        def a_method(self):
            print "parent method"

    class Child(Parent):
        pass

    c = Child()
    print c.an_attribute # "parent attribute"
    c.a_method() # "parent method"

Exceptions to this rule are methods and attributes that start with a double underscore(__) don't get inherited


    class Parent(object):
        __an_attribute = "parent attribute"

        def __a_method(self):
            print "parent method"

    class Child(Parent):
        pass

    c = Child()
    print c.__an_attribute # Error!
    c.__a_method # Error!

Overriding

What happens when an attribute or a method in a child class has the same name as the parent class?

The answer is that the child class will "override" the attribute or method of the parent class

Overriding means that the child's implementation of the attribute or method will replace the implementation of the parent for all instances of that child class

Here is an example of the child overriding some of the parent's attributes


    class Parent(object):
        message = "Hello There!"
        age = 40

    class Child(Parent):
        age = 8
        gender = "male"

    c = Child()
    print c.message # "Hello There!"
    print c.age # Overriden to 8
    print c.gender # "male"

Here is an example of the child overriding a parent's methods


    class Parent(object):
        def get_hobbies(self):
            return ['yoga', 'cooking', 'photography']

    class Child(Parent):
        def get_hobbies(self):
            return ['eating', 'playing', 'pooping']

    c = Child()
    print c.get_hobbies() # ['eating', 'playing', 'pooping']
It doesn't matter if the parameters are different. If a child attribute or method has the same name as the parent, it will replace that attribute or method.

    class Parent(object):
        def receives(self):
            print "I am too old for this $hIt! - Grumpy Parent"

    class Child(Parent):
        def receives(self, item=None):
            if item == "blanket":
                print "ba ba, blankie"
            elif item == "pacifier":
                print "binkie wuv"
            else:
                print "muh muh boo boo"

    c = Child()
    # .receives() is overriden to an implementation that
    # accepts 1 argument
    c.receives("pacifier")

Extending

We already have discussed how to override attributes and methods.
However, sometimes we don't want the child class to override the some of the parent's methods.
We might want to reuse those methods because they can still help.
In that case, we want to "extend" the class
To extend a class, we need to use the super() built-in function
The super() built-in function allows us to call attributes and methods from the parent in the child class
Here is the typical way the super() built-in function is used.

    class Parent(object):
        def method(self, arg):
            print "This is the arg: {}".format(arg)

    class Child(Parent):
        def method(self, arg):
            arg = "{}(called by Child)".format(arg)
            super(Child, self).method(arg)
Let's breakdown the relevant code

    # super(Child, self) evaluates to the parent class
    # (which is named Parent)

    # super(Child, self).method(arg) calls the method named
    # "method" in the Parent Class
                

    c = Child()

    # This prints out "This is the arg: foobar(called by Child)"
    c.method("foobar")
Here is an example of a child class calling a parent attribute

    class Parent(object):
        message = "Don't worry child, Mommy is here"

    class Child(Parent):
        def consult_parent(self):
            return super(Child, self).message

    c = Child()
    print c.consult_parent()
Here is an example of a child class extending a parent method

    class Car(object):
        def get_viable_terrains(self):
            return ['land']

    class JamesBondCar(Car):
        def get_viable_terrains(self):
            parent_viable_terrains = super(
                JamesBondCar,
                self
            ).get_viable_terrains()

            # Super Spy cars can travel anywhere!
            parent_viable_terrains.append('sea')
            parent_viable_terrains.append('air')
            parent_viable_terrains.append('space')

            return parent_viable_terrains

    jbc = JamesBondCar()

    # ['land', 'sea', 'air', 'space']
    print jbc.get_viable_terrains()

Types of inheritance

Single

In Single inheritance, we have exactly one parent and one child
All the examples that we were using uses single inheritance

    class Parent(object):
        pass

    class Child(Parent):
        pass

    c = Child()
    print isinstance(c, Parent) # True

Multilevel

In Multilevel inheritance, we can have more than one parent or child

    class GrandFather(object):
        pass

    class Father(GrandFather):
        pass

    class Child(Father):
        pass

    c = Child()
    print isinstance(c, Father) # True
    print isinstance(c, GrandFather) # True
The parent, the parent's parent, etc. of a class are called ancestor classes

    # An ancestor of Child and Father
    class GrandFather(object):
        pass

    # An ancestor of Child
    class Father(GrandFather):
        pass

    class Child(Father):
        pass
The children, the children's children, etc. of a class are called descendant classes

    class GrandFather(object):
        pass

    # A descendant of GrandFather
    class Father(GrandFather):
        pass

    # A descendant of GrandFather and Father
    class Child(Father):
        pass
Descendant classes will inherit attributes and methods of their ancestor classes, following the rules for overriding and extending classes

    class GrandFather(object):
        age = 70

        def get_playlist(self):
            return ["60's"]

    class Father(GrandFather):
        age = 40

        def get_playlist(self):
            playlist = super(Father, self).get_playlist()
            playlist.append("80's")
            return playlist

    class Child(Father):
        age = 20

        def get_playlist(self):
            playlist = super(Child, self).get_playlist()
            playlist.append("2000's")
            return playlist

    g = GrandFather()
    f = Father()
    c = Child()

    print g.age # 70
    print f.age # 40
    print f.age # 20

    print g.get_playlist() # ["60's"]
    print f.get_playlist() # ["60's", "80's"]
    print c.get_playlist() # ["60's", "80's", "2000's"]
Ancestor classes can be the parent of more than one class

    class Persons(object): pass

    class Employee(Persons): pass
    class Student(Persons): pass

    class Worker(Employee): pass
    class SoftwareDeveloper(Employee): pass

    class SchoolStudent(Student): pass
    class CollegeStudent(Employee): pass

Multiple

In Multiple Inheritance, a child class has more than one parent
To do this, pass the parent names as arguments to the class

    class Father(object):
        pass

    class Mother(object):
        pass

    class Child(Father, Mother):
        pass

    c = Child()
    print isinstance(c, Father) # True
    print isinstance(c, Mother) # True
What happens when a child's parents have similar attributes and/or methods?
Python uses an algorithm called Method Resolution Order(MRO) to decided which attribute or methods get priority
MRO is a complicated algorithm. It takes an long article to explain it in its entirety!
Fortunately, there is a way to circumvent MRO entirely and explicitly define which parent we are referring to.

We do this by using the baseclass syntax


    # baseclass is the name of the parent class

    # attribute refers to the method you want to
    # access in the parent class
    baseclass.attribute

    # method refers to the method you want to
    # access in the parent class
    baseclass.method()
Here is an example demonstrating how to use baseclass

    class Father(object):
        eye_color = 'blue'

        def get_personality_type(self):
            return 'extrovert'

    class Mother(object):
        eye_color = 'green'

        def get_personality_type(self):
            return 'introvert'

    class Child(Father, Mother):
        eye_color = Father.eye_color

        def get_personality_type(self):
            # NOTE: you need to pass self as the first argument
            # to call the base class method using the 'baseclass'
            # style
            return Mother.get_personality_type(self)

    c = Child()
    print c.eye_color
    print c.get_personality_type()

To recap:

use super() for single/multilevel inheritance, baseclass.method() for multiple inheritance

A word of caution

Multiple inheritance is a very powerful feature
However, this kind of inheritance is complex and error prone.
This is especially true if you are using multiple inheritance with many classes

You should always ask yourself if you can implement what you had in mind using single or multilevel inheritance

Because there's almost always a alternative solution that doesn't involve using multiple inheritance.
